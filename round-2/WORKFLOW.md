# WORKFLOW

# FE-03 – AI-Assisted Workflow Drill

## Objective

The objective of this assignment was to compare two different AI-assisted development workflows by implementing the same feature twice.

The selected feature for this assignment was a **Login & Signup Authentication Form** built using HTML, CSS, and Vanilla JavaScript.

---

# Development Process

## Round 1 – Vague Prompt

### Prompt

```text
Create a login and signup page.
```

### Workflow

1. Provided a short and generic prompt to the AI.
2. Accepted the generated implementation with minimal guidance.
3. Performed only basic review of the generated code.
4. Saved the implementation as the baseline version.

### Result

The AI produced a functional authentication interface quickly, but several improvements were still required, including better validation, accessibility, and maintainability.

---

# Round 2 – Structured Prompt

### Workflow

The AI followed a structured development process before writing code.

The workflow included:

1. Reviewing the existing project structure.
2. Creating a detailed implementation plan.
3. Defining file responsibilities.
4. Building semantic HTML.
5. Applying responsive CSS.
6. Implementing client-side validation.
7. Improving accessibility.
8. Handling common edge cases.
9. Performing verification and self-review.
10. Correcting identified issues before completion.

---

# AI Review and Improvements

During verification, the AI identified and corrected several issues:

- Missing JavaScript file reference.
- Incorrect HTML and JavaScript selector mapping.
- Password visibility toggle behavior.
- Login and Signup form switching.
- Inline validation improvements.
- Accessibility enhancements.

---

# Comparison

| Round 1 | Round 2 |
|---------|---------|
| Vague prompt | Structured prompt |
| Minimal planning | Detailed planning |
| Basic implementation | Organized implementation |
| Limited validation | Complete client-side validation |
| Basic review | AI self-review and verification |
| More manual corrections | Fewer manual corrections |
| Baseline implementation | Production-quality implementation |

---

# Key Learnings

- Prompt quality directly affects AI-generated results.
- Planning before coding improves overall development quality.
- AI-generated code should always be reviewed and verified.
- Breaking development into phases makes implementation easier to manage.
- Structured workflows produce cleaner and more maintainable code.

---

# Conclusion

This assignment demonstrates how a structured AI-assisted workflow produces significantly better results than relying on a vague prompt alone.

By comparing both approaches, it becomes clear that planning, verification, and iterative refinement lead to higher-quality frontend implementations and a more professional development process.
