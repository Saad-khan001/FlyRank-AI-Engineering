# FE-03: AI-Assisted Workflow Drill

## Assignment Overview

This assignment is part of the **FlyRank AI Engineering Internship** and focuses on understanding how different prompting strategies influence the quality of AI-generated code.

The objective is to build the same frontend feature twice using two different AI-assisted workflows and compare the outcomes.

---

## Assignment Objective

The primary goal of this assignment is to evaluate the impact of prompt quality on software development by comparing:

- **Round 1:** A basic implementation created using a single vague AI prompt.
- **Round 2:** An improved implementation created using a structured AI prompt with planning, constraints, verification, and documentation.

---

## Feature Implemented

**Login & Signup Authentication Interface**

The project demonstrates a responsive authentication interface developed through two different AI-assisted development approaches.

---

## Assignment Workflow

### Round 1 – Basic AI Workflow

- Single vague prompt
- Minimal planning
- Direct AI-generated implementation
- Baseline result for comparison

### Round 2 – Structured AI Workflow

- Detailed prompt
- Planning before development
- Step-by-step implementation
- Code verification
- Manual testing
- Documentation
- Workflow refinement

---

## Repository Organization

This repository is organized into separate branches for each implementation:

- **feature/round-1** – Basic AI-generated implementation
- **feature/round-2** – Structured AI-assisted implementation

Each branch contains its own source code, documentation, and supporting files relevant to that workflow.

---

## Learning Outcome

Through this assignment, I learned how structured prompting, planning, testing, and documentation significantly improve the quality, maintainability, and reliability of AI-assisted software development compared to using a single vague prompt.
